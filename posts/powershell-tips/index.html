Powershell Tips

October 3, 2020 3-minute read
Software • powershell

Command Line i.e cmd.exe are still in use in the today system, largely because there are deep entanglements of batch processes in the legacy (old) system.
PowerShell is coming offering much more of the old command line in a more standard way. With standardization, it can provide homogeneity and best practices for the advanced user.

For me, these 2 things are big,

  1. Piping, for output redirection
  2. Alias, for make it Linux like commands

We can get the recent version from PowerShell’s Github release page.

We can notice from the alias, the dir and ls are the same, they are listing items (Get-ChildItem).
It is convenient to navigate the system structure using command alias that we already familiar with.

PS: Environment Variable

Print path

ls env:

Another use of an alias, the same output will work with Get-ChildItem env:.
Then, from the list we can going on the path detail,

$env:path
PS: Pipeline

Use pipeline, listing items pipe sort descending.

ls | sort -Descending
PS: List Processes
Get-Process

Filter the processes by name, I want to find Hugo and VS-code processes.

Get-Process -Name hugo, code

PS: Help
Get-Help

Help on Get-Process, use alias help -> Get-Help Get-Process -> gps

help gps

Closing

As shown with Alias and Pipeline, Powershell is a new tool that fit into my daily usage, Gone is the context switching back-of-our-mind where we need to pause and re-assess, are we working on Linux or Windows?
With the Linux command line like (Alias!), it is easier to navigate the system with common commands.
